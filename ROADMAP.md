# ROADMAP — План развития qkbn

## Обзор

Этот документ описывает план развития утилиты **qkbn** — локального веб-сервера с Kanban-доской для визуализации сессий Qwen-code.

---

## v0.2.0 — Гибкая конфигурация запуска

### 1. Флаг для настройки порта (`--port`)

**Цель:** Позволить пользователю указывать порт веб-сервера через CLI-флаг.

**Требования:**
- Флаг `--port` (или `-p`) с типом `int`
- Значение по умолчанию: `9090`
- Валидация диапазона портов (1–65535)
- Вывод ошибки при недоступности порта

**Пример использования:**
```bash
qkbn --port 8080
qkbn -p 3000
```

**Критерии приемки:**
- [ ] Флаг парсится через `flag.IntVar`
- [ ] При невалидном порте выводится понятная ошибка
- [ ] Порт логгируется при старте сервера
- [ ] Покрыто unit-тестами (валидация, значение по умолчанию)

---

### 2. Флаг для настройки директории с задачами (`--todos-dir`)

**Цель:** Позволить указывать кастомную директорию с JSON-файлами задач.

**Требования:**
- Флаг `--todos-dir` (или `-d`) с типом `string`
- Значение по умолчанию: `~/.qwen/todos/`
- Поддержка `~` в пути (расширение до домашней директории)
- Проверка существования директории при старте
- Вывод ошибки если директория недоступна

**Пример использования:**
```bash
qkbn --todos-dir /custom/path/to/todos
qkbn -d ~/my-todos
```

**Критерии приемки:**
- [ ] Флаг парсится через `flag.StringVar`
- [ ] Функция `expandPath` обрабатывает `~`
- [ ] Проверка директории до запуска сервера
- [ ] Покрыто unit-тестами (расширение пути, несуществующая директория)

## v0.3.0 — Модульная архитектура UI

### 3. Вынос веб-интерфейса в отдельный пакет

**Цель:** Отделить логику HTTP-сервера и шаблонов от `main.go` для лучшей поддерживаемости.

**Требования:**
- Создать пакет `internal/http` или `pkg/http`
- Перенести `kanbanHandler` в отдельный файл
- Вынести HTML-шаблон в `templates/kanban.html`
- Использовать `embed.FS` для встраивания шаблона **при запуске** (не при сборке)
- Сохранить возможность сборки в один бинарник

**Структура после изменений:**
```
qkbn/
├── cmd/qkbn/
│   └── main.go          # Только CLI и инициализация
├── internal/http/
│   └── server.go        # HTTP-сервер и хендлеры
├── templates/
│   └── kanban.html      # HTML-шаблон доски
└── ...
```

**Важно:**
- Шаблон должен встраиваться в бинарник через `//go:embed`
- Это происходит **на этапе компиляции**, но используется **при запуске**
- Не требуется внешних файлов после сборки

**Критерии приемки:**
- [ ] `main.go` не содержит HTML-шаблонов
- [ ] Хендлер вынесен в отдельный пакет
- [ ] Шаблон используется через `embed.FS`
- [ ] Сборка работает без ошибок
- [ ] Покрыто интеграционными тестами (HTTP-запросы)

## v0.4.0 — Поддержка множественных сессий

### 4. Отображение всех сессий Qwen-code

**Цель:** Показывать все задачи из всех JSON-файлов в директории, с группировкой по сессиям/проектам.

**Проблема:**
Сейчас читается только **один** файл (последний изменённый). Пользователь может иметь несколько активных сессий Qwen-code, но видит только одну.

**Решение:**
- Читать **все** JSON-файлы из директории
- Группировать задачи по источнику (сессии/проекту)
- Отображать на доске с визуальным разделением

**Варианты реализации:**

#### Вариант A: Группировка по файлам (сессиям)
- Каждая сессия — отдельная "карточка" на доске
- Заголовок группы = имя файла или название проекта
- Переключение между группами через dropdown или вкладки

#### Вариант B: Сводная доска с метками
- Все задачи в общих колонках
- Цветовая маркировка по сессиям
- Фильтр по сессии (опционально)

#### Вариант C: Иерархическая доска
- Слева — список сессий
- Справа — Kanban для выбранной сессии
- Опция "показать все" для сводного вида

**Требования:**
- Чтение всех `*.json` файлов из директории
- Парсинг каждого файла в общую структуру
- Группировка по полю `sessionName` или имени файла
- UI с визуальным разделением групп
- Сортировка: активные сессии сверху

**Пример структуры данных:**
```go
type Session struct {
    Name   string
    Tasks  []Task
    Updated time.Time
}

type Board struct {
    Sessions []Session
    Todo     []Task // aggregated
    InProgress []Task
    Done     []Task
}
```

**Критерии приемки:**
- [ ] Читаются все JSON-файлы из директории
- [ ] Задачи группируются по сессиям
- [ ] UI отображает название сессии для каждой задачи
- [ ] Добавлена сортировка сессий (по времени обновления)
- [ ] Покрыто unit-тестами (чтение нескольких файлов, пустая директория)
- [ ] Интеграционные тесты для множественных сессий

## Технические заметки

### Embed vs Runtime
- `//go:embed` работает **на этапе компиляции**
- Файлы встраиваются в бинарник
- При запуске не требуется доступ к файловой системе
- Это соответствует требованию "цеплять при запуске"

### Обратная совместимость
- Флаги должны быть опциональными
- Значения по умолчанию сохраняют текущее поведение
- Старые команды должны работать без изменений

### Тестирование
- Unit-тесты: парсинг флагов, валидация, группировка задач
- Интеграционные: HTTP-запросы, рендеринг шаблона
- Coverage цель: >60%

## v0.6.0 — Современный Web UI с навигацией и группировкой

### 5. Редизайн Web UI с навигационной панелью и вкладками

**Цель:** Создать современный, визуально привлекательный интерфейс с удобной навигацией и группировкой сессий по статусу активности.

**Проблема:**
Текущий UI имеет простой дизайн без навигации. Все сессии отображаются списком без группировки по статусу активности, что затрудняет восприятие при большом количестве сессий.

**Решение:**
- Добавить навигационную панель (navbar) с логотипом и вкладками
- Группировать сессии по трём категориям: Активные / Неактивные / Завершенные
- Переключение между категориями через вкладки (без перезагрузки страницы)
- Современный дизайн с использованием CSS-переменных и анимаций

**Требования:**

#### 5.1 Навигационная панель (Navbar)
- **Логотип:** Квадратная иконка с буквой "Q" на синем фоне
- **Название:** "Qwen-code" кликабельное, ведёт на главную
- **Вкладки:**
  - 🔥 Активные — сессии с pending/in_progress задачами
  - ⏸️ Неактивные — сессии только с pending задачами (нет in_progress)
  - ✅ Завершенные — сессии только с completed задачами
- **Стиль:**
  - Sticky-позиционирование (фиксированная при скролле)
  - Полупрозрачный фон с blur-эффектом (backdrop-filter)
  - Подчёркивание активной вкладки синим цветом

#### 5.2 Группировка сессий
- **Логика группировки:**
  - `Active` — есть pending ИЛИ in_progress задачи
  - `Inactive` — есть только pending задачи (без in_progress)
  - `Completed` — есть только completed задачи

- **Визуальное оформление сессий:**
  - Активные: зелёная левая граница (4px), pulse-индикатор "В работе"
  - Неактивные: серая левая граница, бейдж "Ожидает"
  - Завершенные: фиолетовая левая граница, бейдж "Выполнено", reduced opacity

#### 5.3 Улучшения карточек задач
- **Счётчики задач:** В заголовке каждой колонки отображать количество задач
- **Кнопка "Новая задача":** Внизу колонки TODO кнопка с подсказкой про команду `todowrite`
- **Hover-эффекты:** При наведении карточка приподнимается (transform: translateY)
- **Стили для completed задач:** Зачёркнутый текст, reduced opacity

#### 5.4 JavaScript функциональность
- **Переключение вкладок:**
  - Скрывать все секции по умолчанию
  - Показывать только активную секцию
  - Переключение без перезагрузки страницы
- **Сохранение автообновления:** Meta refresh для обновления данных каждые N секунд

#### 5.5 Адаптивный дизайн
- **Desktop:** 3 колонки в ряд
- **Tablet/Mobile:** 1 колонка (вертикальный список)
- **Navbar:** На мобильных складывается в вертикальный список

**Структура данных:**
```go
type PageData struct {
    Sessions          []SessionData
    ActiveSessions    []SessionData  // pending OR in_progress
    InactiveSessions  []SessionData  // only pending, no in_progress
    CompletedSessions []SessionData  // only completed
    UIRefreshInterval int
}
```

**CSS-переменные:**
```css
:root {
    --active-color: #10b981;      /* зелёный для активных */
    --inactive-color: #94a3b8;    /* серый для неактивных */
    --completed-color: #8b5cf6;   /* фиолетовый для завершенных */
    --primary-color: #2563eb;     /* синий для акцентов */
}
```

**Критерии приемки:**
- [ ] Навигационная панель отображается корректно
- [ ] Вкладки переключают секции без перезагрузки
- [ ] Сессии правильно группируются по статусу
- [ ] Pulse-индикатор анимируется для активных сессий
- [ ] Счётчики задач отображаются в заголовках колонок
- [ ] Кнопка "Новая задача" отображается в колонке TODO
- [ ] Адаптивный дизайн работает на узких экранах
- [ ] Автообновление страницы работает (meta refresh)
- [ ] Покрыто интеграционными тестами (рендеринг шаблона)

## v0.6.1 — Улучшение Web UI: SPA архитектура и полный функционал

### 6. Превращение в SPA (Single Page Application)

**Цель:** Убрать полную перезагрузку страницы, обновлять только изменившиеся данные.

**Проблема:**
Сейчас используется `<meta http-equiv="refresh">`, что вызывает полную перезагрузку страницы каждые N секунд. Это приводит к:
- Миганию экрана
- Сбросу состояния JavaScript (позиция скролла, открытые вкладки)
- Излишней нагрузке на браузер

**Решение:**
- Удалить meta refresh
- Добавить JavaScript для периодических fetch-запросов
- Обновлять только содержимое секций (DOM-элементы)
- Сохранять состояние вкладок при обновлении

**Требования:**

#### 6.1 API endpoint для данных
- Создать новый хендлер `/api/sessions` возвращающий JSON
- Формат ответа:
```json
{
  "activeSessions": [...],
  "inactiveSessions": [...],
  "completedSessions": [...],
  "lastUpdated": "2026-03-12T17:00:00Z"
}
```

#### 6.2 JavaScript для обновлений
- Fetch-запрос каждые N секунд (настраиваемый интервал)
- Обновление DOM только для изменившихся секций
- Индикатор последнего обновления
- Обработка ошибок сети (retry logic)

#### 6.3 Оптимизация рендеринга
- Кэширование данных на клиенте
- Сравнение версий (ETag или timestamp)
- Частичное обновление (только изменившиеся карточки)

**Критерии приемки:**
- [ ] Страница не перезагружается полностью
- [ ] Данные обновляются каждые N секунд через fetch
- [ ] Состояние вкладок сохраняется при обновлении
- [ ] Позиция скролла не сбрасывается
- [ ] Индикатор последнего обновления отображается
- [ ] Обработка ошибок сети (показать уведомление)

### 7. Исправление отображения всех секций

**Цель:** Отображать данные во всех вкладках (Активные/Неактивные/Завершенные).

**Проблема:**
Сейчас в `refreshSessionsCache()` completed-сессии фильтруются и не попадают в кэш. Поэтому вкладка "Завершенные" пустая.

**Решение:**
- Убрать фильтрацию completed-сессий в `refreshSessionsCache()`
- Хранить все сессии в кэше
- Группировать по статусу в `KanbanHandler()`

**Изменения в коде:**
```go
// Было:
if status == "completed" {
    continue // Пропускаем
}

// Стало:
// Оставляем все сессии, группировка будет в хендлере
```

**Критерии приемки:**
- [ ] Completed-сессии сохраняются в кэш
- [ ] Вкладка "Завершенные" отображает данные
- [ ] Вкладка "Неактивные" отображает данные
- [ ] Все три вкладки работают корректно

**Критерии приемки:**
- [ ] Кнопка кликабельна

- [ ] Кнопка копирования команды в буфер
