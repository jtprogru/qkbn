# QWEN.md — Контекст проекта

## Обзор проекта

**qkbn** — это утилита на Go, которая запускает локальный веб-сервер с Kanban-доской для визуализации активных сессий Qwen-code. Сервер читает JSON-файлы с задачами из директории `~/.qwen/todos/` и отображает их в виде трёх колонок: TODO, IN PROGRESS, DONE.

### Основные возможности

- Автоматическое обновление страницы каждые 5 секунд
- Чтение последнего изменённого JSON-файла с задачами
- Группировка задач по статусу (`pending`, `in_progress`, `completed`)
- Отображение описания задачи и текущего действия

### Технологии

- **Язык:** Go 1.25.0
- **Веб-сервер:** стандартный `net/http`
- **Шаблонизация:** `html/template`
- **Сборка:** Taskfile + GoReleaser

## Сборка и запуск

### Требования

- Go 1.25.0+
- [Task](https://taskfile.dev/) (опционально, для удобства)
- golangci-lint (для линтинга)

### Команды

Все команды доступны через `Taskfile.yml`:

```bash
# Запуск через go run
task run

# Сборка бинарника (в ./dist/)
task build

# Установка глобально
task install

# Форматирование кода
task fmt

# Линтинг
task lint

# Запуск всех тестов
task test

# Тесты с покрытием (генерирует cover.html)
task test:coverage

# Очистка артефактов
task clean

# Создание снапшот-релиза
task release
```

### Ручной запуск

```bash
# Запуск
go run cmd/qkbn/main.go

# Сборка
CGO_ENABLED=0 go build -o ./dist/qkbn cmd/qkbn/main.go

# Установка
go install ./cmd/qkbn
```

## Использование

После запуска сервер доступен по адресу **http://localhost:9090**.

Сервер читает файлы из `~/.qwen/todos/` (формат JSON):

```json
{
  "todos": [
    {
      "content": "Описание задачи",
      "activeForm": "Текущее действие",
      "status": "pending"
    }
  ]
}
```

Статусы задач:
- `pending` — колонка TODO
- `in_progress` — колонка IN PROGRESS
- `completed` — колонка DONE

## Разработка

### Линтинг

Проект использует `golangci-lint` с расширенным набором линтеров (почти все включены, некоторые отключены или настроены через исключения).

```bash
golangci-lint run -v
```

### Стиль кода

- Форматирование: `gofmt`
- Импорты: группировка через `gci` (standard → default → local)
- Обработка ошибок: явная, через `if err != nil`
- Именование: в стиле Go (CamelCase для экспортируемых, snake_case для внутренних)

### Тестирование

На момент анализа тесты отсутствуют. При добавлении нового функционала рекомендуется покрывать тестами критическую логику.
